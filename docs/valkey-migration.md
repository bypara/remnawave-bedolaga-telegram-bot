# Переход с Redis 7 на Valkey 9

С версии, содержащей это изменение, `docker-compose.yml` использует
[Valkey](https://valkey.io) (`valkey/valkey:9-alpine`) вместо `redis:7-alpine`.

## Зачем

**Лицензия.** Тег `redis:7-alpine` уже давно указывает на Redis 7.4, который
распространяется под двойной лицензией RSALv2/SSPLv1 — ни одна из них
не является open-source по определению OSI. (В Redis 8 добавлена опция
AGPLv3 — она OSI-approved, но это сильный copyleft.) Valkey — форк Redis 7.2.4 под
BSD-3-Clause, развиваемый под крылом Linux Foundation (AWS, Google, Oracle,
Ericsson); именно им заменили Redis в репозиториях Fedora, Debian, Alpine
и openSUSE.

**Производительность.** Valkey 8/9 получили асинхронный многопоточный I/O
(до ~3× пропускной способности на многоядерных машинах по замерам upstream),
новую hash-таблицу (экономия ~20+ байт на ключ — наш `maxmemory 256mb`
вмещает больше), ускоренную репликацию и hash field expiration.

**Совместимость.** Протокол RESP2/RESP3 и команды идентичны. Бот работает
через `redis-py` (`redis.asyncio`) по `REDIS_URL` — ни одной строчки кода
менять не нужно, схема URL остаётся `redis://`. Имя сервиса, контейнера и
тома в compose тоже намеренно не менялись.

## Новые установки

Ничего делать не нужно — `docker compose up -d` сразу поднимет Valkey.

## Обновление существующей установки

Redis в этом боте хранит только эфемерные данные: FSM-состояния диалогов,
кэши, rate-limit-счётчики, отложенные deep-link payload'ы и сохранённые
корзины. Подписки, балансы, платежи и пользователи живут в PostgreSQL и
не затрагиваются.

AOF/RDB-файлы Redis 7.4 несовместимы с Valkey (форматы разошлись после
форка на 7.2), поэтому самый надёжный путь — пересоздать том:

```bash
docker compose stop bot redis
docker compose rm -f redis
docker volume rm <имя_проекта>_redis_data
docker compose up -d
```

Что при этом произойдёт (одноразово):

- пользователи, находившиеся в середине диалога с ботом, начнут его заново
  (`/start`);
- сохранённые корзины очистятся — товары придётся добавить заново;
- кэши и rate-limit-счётчики прогреются с нуля.

Если ваш том создавался ещё на Redis ≤ 7.2, Valkey загрузит его данные
напрямую — можно обойтись `docker compose pull redis && docker compose up -d redis`,
но вариант с пересозданием тома работает в любом случае.
